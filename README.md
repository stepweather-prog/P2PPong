# P2PPong (HTTPR Engine v6.3)

[![License: MIT](https://shields.io)](https://opensource.org)
[![Protocol Specification](https://shields.io)](#-криптографический-монолит)

**P2PPong** — это транспортно-агностическое криптографическое ядро (работающее через Web Crypto API и Web Workers), которое является физическим воплощением протокола **HTTPR (HTTP Relay/Router)**. 

Протокол спроектирован по принципам ОГАС академика В.М. Глушкова. Он не использует сторонние библиотеки, работает прямо в браузере и относится к серверам (Cloudflare Workers, Render, Firebase) как к временному расходному материалу для слепой маршрутизации.

---

## 🏗️ Архитектура единого ядра (HTTPR)

Ядро P2PPong объединяет в себе криптографию, дедупликацию и управление сетевыми эшелонами без промежуточных слоев:

1. **Эшелон Cloudflare (Основной):** Быстрый REST API для ведения анонимного пула маяков и рандеву.
2. **Эшелон Render (Резервный):** Горячий бэкенд-дублер на случай блокировки или падения основного эшелона.
3. **Эшелон WebRTC (Прямой P2P):** Чистый DataChannel между браузерами, устанавливаемый сразу после успешного HTTPR-рукопожатия.

---

## 🔒 Криптографический монолит

Ядро реализует бескомпромиссную модель приватности (Blind Locker), полностью скрывая метаданные от промежуточных серверов-ретрансляторов:

* **Triple Ratchet (Signal-совместимый):** Symmetric Ratchet для каждого пакета + DH Ratchet (на кривой P-256) каждые 10 сообщений для обеспечения *Post-Compromise Security*. Архитектура включает алгоритм авто-ресинхронизации (`advanceRecvRatchet`).
* **Конвертное шифрование HTTPR (AES-256-GCM):** Внешний транспортный слой шифрует конверт. Снаружи видны только `v`, `tid`, `hop`, `ttl`, `ts` и `beaconId`. Сервер не знает, кто отправитель и кто получатель.
* **Perfect Forward Secrecy:** Ключи отправки уничтожаются сразу после использования. Старые входящие ключи хранятся в кэше глубиной в 3 шага для сборки пакетов, пришедших со случайной задержкой.
* **Анти-флуд и Целостность:** Каждый блоб подписывается **HMAC-SHA256** и снабжается монотонно растущим индексом `ri`. Пакеты с устаревшим или дублирующимся индексом отбрасываются.
* **Маскировка трафика:** Перед отправкой применяется GZIP-сжатие (`CompressionStream API`) и добавляется случайный паддинг от 20 до 70 байт для разрушения сигнатур анализа размера пакетов (Traffic Shaping).

## 🧩 Дедупликация и Фрагментация

* **Авто-фрагментация:** Ядро берет тяжелые пакеты (например, голосовые сообщения до 50 КБ / 10 секунд) и автоматически нарезает их на мелкие фрагменты для гарантированной пропускной способности через HTTP-опрос (Polling). На стороне получателя происходит прозрачная сборка.
* **Дедупликация:** Каждый пакет имеет уникальный транспортный ID (`tid`). Кэш глубиной в 1000 записей предотвращает повторную обработку дубликатов, если один и тот же пакет долетел через разные эшелоны связи.

## 🎯 Режимы маяков (Колчаны)

* **Обычный маяк:** Адресная ячейка `waiting_<beaconId>`. Идентификатор передается собеседнику напрямую.
* **Публичный колчан (Blind Pool):** Маяк публикуется в анонимный общий пул `/pool`. Собеседник скачивает весь пул (до 100 штук) и «вслепую» перебирает их своими ключами. Сервер не может сопоставить участников.
* **Тайный колчан:** Идентификатор вычисляется как `SHA-256(секрет + соль)`. Требует предварительного безопасного обмена текстовым секретом.

## 🚀 Основные методы API (`p2ppong.js`)

``javascript
// Инициализация криптографического движка и Web Worker
await P2PPong.init();

// Создание маяков разных типов
const beaconId = P2PPong.craftArrow();
const publicBeaconId = P2PPong.craftPublicArrow();
const secretBeaconId = P2PPong.craftSecretArrow("мой_тайный_пароль");

// Подключение к пиру
P2PPong.joinBeacon(targetBeaconId);
P2PPong.joinPublicPool();

// Подтверждение 7-значного кода верификации (23 бит энтропии)
P2PPong.confirmVerification();

// Отправка данных (автоматически упаковывается в Triple Ratchet и HTTPR-конверт)
P2PPong.sendMessage(channelId, "Привет, RobinHood!");
P2PPong.sendVoiceMessage(channelId, audioBase64); // Лимит 50 КБ
---



 📋 Системные требования
Любой современный браузер (или среда PWA) с поддержкой:
* `Web Crypto API` (`SubtleCrypto` для P-256, AES-GCM, HMAC)
* `Web Workers` (для выноса криптографии из основного потока интерфейса)
* `CompressionStream API` (GZIP-сжатие)
