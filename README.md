# P2PPong v1.0 — Распределённая P2P-платформа

Децентрализованная платформа для защищённой P2P-коммуникации, построенная на принципах ОГАС академика Глушкова. Работает в браузере без установки, без центрального сервера хранения данных.

## Архитектура

│ ПРИЛОЖЕНИЯ (на платформе) │
│ - RobinHood (мессенджер) │
│ - Учёт ресурсов / Электронные чеки / Аукционы │
├─────────────────────────────────────────────────────────┤
│ P2PPong (ядро) │
│ - Событийная шина (on/emit) │
│ - Криптография (ECDH + AES-GCM + Double Ratchet) │
│ - DHT (Kademlia, 256 корзин, k=20, α=3) │
│ - Маяки (blind beacon, адресная доставка) │
├─────────────────────────────────────────────────────────┤
│ ТРАНСПОРТ │

│ - HTTP Polling (Render) — резервный │

## Три уровня сигнальной сети (ОГАС)

| Уровень | Технология | Назначение |
|---------|------------|------------|
| Главный центр | Cloudflare Worker (WebSocket) | Быстрая адресная маршрутизация маяков |
| Опорные центры | Render HTTP Server (polling) | Резервный канал при блокировке WebSocket |
| Терминалы | Пиры в браузерах (DHT + WebRTC) | Прямые соединения без серверов |

## Слепой Улей

Серверы не читают содержимое сообщений, не хранят историю, не знают кто с кем общается. Только пересылают зашифрованные блобы конкретному адресату. Никакого broadcast.

## Идентификация

**Peer ID** = первые 32 символа SHA-256 от:
- Параметров WebGL-рендерера (вендор + модель GPU)
- Характеристик AudioContext (частота дискретизации, число каналов)
- Размеров экрана и глубины цвета
- `navigator.hardwareConcurrency` и `deviceMemory`
- Случайной соли (генерируется однократно)

Стабильный, но не привязанный к личности идентификатор устройства.

## Сквозное шифрование (E2EE)

### Рукопожатие
ECDH на кривой P-256. Каждая сторона генерирует ephemeral-пару, обменивается публичными ключами через маяки (blind beacon).

### Double Ratchet + HMAC
- Ratchet Key обновляется **каждое сообщение**: `SHA-256(старый_ключ + индекс)`
- HMAC-SHA256 подписывает каждое сообщение
- Старые ключи сохраняются (до **50**) для расшифровки сообщений, пришедших не по порядку
- Ресинхронизация: при расхождении ratchet — автоматический перезапуск через ECDH

### Шифрование сообщений
- **AES-256-GCM**, случайный IV (12 байт)
- **GZIP-сжатие** перед шифрованием
- **Случайный паддинг** (20–70 байт)
- **Фиксированный размер блоба: 4096 байт** (padding против анализа размера)
- Защита от replay-атак (проверка `ratchetIndex`)

## Маяки и каналы

| Сущность | Время жизни | Описание |
|----------|-------------|----------|
| Маяк (beacon) | 5 минут | Зашифрованное объявление с `targetPeerId` |
| Канал (channel) | 10 минут (продлевается) | Зашифрованный туннель между двумя пирами |

## Событийная шина

Ядро **не зависит от интерфейса**. Всё общение через события:

```javascript
// Подписка
P2PPong.on('message-received', (data) => { ... });
P2PPong.on('channel-opened', (data) => { ... });
P2PPong.on('beacon-received', (data) => { ... });

// Отправка
P2PPong.sendMessage(channelId, 'привет');
P2PPong.createBeacon(targetPeerId, { nick: 'Алиса' });
