# P2PPong v1.0 — Распределённая P2P-платформа

Децентрализованная платформа для защищённой P2P-коммуникации, построенная на принципах ОГАС академика Глушкова. Работает в браузере без установки, без центрального сервера хранения данных.

--

## Архитектура

Платформа состоит из трёх слоёв:

**Приложения (верхний слой)**
- RobinHood (мессенджер), учёт ресурсов, электронные чеки, аукционы

**P2PPong (ядро)**
- Событийная шина: `P2PPong.on()` / `P2PPong.send()`
- Криптография: ECDH + AES-256-GCM + Double Ratchet
- DHT: Kademlia, 256 корзин, k=20, α=3
- Маяки: blind beacon, адресная доставка

**Транспорт (нижний слой)**
- WebSocket (Cloudflare Worker) — основной
- HTTP Polling (Render) — резервный
- WebRTC DataChannel — прямой DHT

--

## Три уровня сигнальной сети (ОГАС)

|      Уровень   |             Технология         |          Назначение             |
|                |                                |                                 |
|  Главный центр | Cloudflare Worker (WebSocket)  | Адресная маршрутизация маяков   |
| Опорные центры | Render HTTP Server (polling)   | Резерв при блокировке WebSocket |
|  Терминалы     | Пиры в браузерах (DHT + WebRTC)| Прямые соединения без серверов  |

--

## Слепой Улей

Серверы не читают сообщения, не хранят историю, не знают кто с кем общается. Только пересылают зашифрованные блобы адресату. **Никакого broadcast.**

--

## Идентификация

**Peer ID** = первые 32 символа SHA-256 от:
- Параметров WebGL (вендор + модель GPU)
- AudioContext (частота, каналы)
- Размеров экрана, глубины цвета
- `navigator.hardwareConcurrency`, `deviceMemory`
- Случайной соли (однократно, localStorage)

Стабильный, не привязан к личности.

--

## E2EE (сквозное шифрование)

### Рукопожатие
ECDH P-256. Ephemeral-пары. Обмен ключами через маяки.

### Double Ratchet + HMAC
- Ключ обновляется каждое сообщение: `SHA-256(ключ + индекс)`
- HMAC-SHA256 на каждое сообщение
- 50 старых ключей для порядка
- Авторесинхронизация при расхождении

### Шифрование
- AES-256-GCM, случайный IV (12 байт)
- GZIP-сжатие
- Случайный паддинг 20–70 байт
- Фиксированный блоб: 4096 байт
- Защита от replay-атак
