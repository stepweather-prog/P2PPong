P2PPong v1.0 — Распределённая P2P-платформа
Версия протокола: 1.0. Кривая: P-256. Шифрование: AES-256-GCM. HMAC: SHA-256. При смене кривой или алгоритма шифрования — мажорное обновление версии протокола.

Децентрализованная платформа для защищённой P2P-коммуникации, построенная на принципах ОГАС академика Глушкова. Работает в браузере без установки, без центрального сервера хранения данных.

Модель угроз
Платформа защищает от:

Чтения сообщений третьей стороной (AES-256-GCM + ECDH P-256 + Double Ratchet)

Подмены сообщений (HMAC-SHA256 на каждом пакете)

Повторного воспроизведения старых сообщений (одноразовые nonce + ratchet)

Раскрытия истории при компрометации ключа (forward secrecy — sending-ключи не сохраняются)

Связывания сессий по Peer ID (beaconId скрывает идентификатор от сервера)

Массового сбора метаданных в публичном пуле (сервер не знает, какой маяк кому принадлежит)

Платформа НЕ защищает от:

Корреляции пиров по IP-адресам и времени запросов (сервер видит, что два IP обратились к одному ключу в близкое время; в режиме публичного пула корреляция ослаблена)

MitM при компрометации сигнального сервера (код верификации передаётся через сервер, не вне полосы; рекомендуется подтверждать код голосом или при встрече)

Анализа паттернов трафика (частота и размер пакетов видны сетевому наблюдателю)

Атак на конечные устройства (вредоносное ПО, кейлоггеры, захват браузера)

Рассинхрона ratchet при одновременной отправке сообщений обеими сторонами

Слепой Улей
Серверы не читают сообщения, не хранят историю, не знают кто с кем общается. Только пересылают зашифрованные блобы адресату. Никакого broadcast. Peer ID скрыт от сервера за beaconId. В режиме публичного пула сервер не знает, какой маяк кому принадлежит.

Идентификация
Peer ID = 32 случайных hex-символа (криптографически безопасный PRNG). Генерируется заново при каждом соединении. Не привязан к устройству, не сохраняется между сессиями.

Beacon ID = случайный идентификатор маяка. Используется как ключ на сигнальном сервере. Peer ID скрыт от сервера. Передаётся между пользователями для соединения.

Режимы соединения
Обычный маяк — beaconId как ключ на сервере. Peer ID скрыт.

Публичный колчан — маяк в общем пуле /pool. Джойнер скачивает все маяки и расшифровывает. Сервер не знает, какой маяк кому принадлежит.

Тайный колчан — beaconId = SHA-256(секрет + соль). Только создатель и джойнер могут вычислить ключ.

E2EE (сквозное шифрование)
Рукопожатие
ECDH P-256. Ephemeral-пары. Обмен ключами через маяки. Ключ маяка = SHA-256(pubKey + 'beacon'). Nonce удалён из открытой части пакета.

Раздельные Ratchet + HMAC
sendKey/sendIndex — для исходящих сообщений

recvKey/recvIndex — для входящих сообщений

Sending-ключ обновляется каждое сообщение: SHA-256(sendKey + 'send_' + sendIndex)

Receiving-ключ прокручивается до нужного индекса при получении

HMAC-SHA256 на каждое сообщение

3 старых receiving-ключа для восстановления порядка

Sending-ключи не сохраняются (forward secrecy)

Авторесинхронизация при расхождении

Шифрование
AES-256-GCM, случайный IV (12 байт)

GZIP-сжатие

Случайный паддинг 20–70 байт

Максимальный размер пакета: 65536 байт (серверный лимит)

Защита от replay-атак через одноразовые nonce

Верификация
7-значный цифровой код (crypto.getRandomValues). Подтверждение: ввод кода + опционально QR / голос. При совпадении канал открывается автоматически.

Маяки и каналы
Сущность	TTL	Описание
Маяк	5 мин	Зашифрованное объявление с beaconId
Маяк в пуле	5 мин	Маяк в общем пуле /pool (до 100 маяков)
Канал	10 мин	Защищённый туннель между пирами
Событийная шина
// Инициализация
P2PPong.init()

// Создание маяков
P2PPong.craftArrow()                  // обычный маяк, возвращает beaconId
P2PPong.craftPublicArrow()            // маяк в общий пул
P2PPong.craftSecretArrow(secret)      // тайный маяк с хэшем от секрета

// Присоединение
P2PPong.joinBeacon(targetBeaconId)    // к обычному маяку
P2PPong.joinPublicPool()              // найти маяк в общем пуле
P2PPong.joinSecretBeacon(secret, salt) // к тайному маяку

// Верификация
P2PPong.confirmVerification()         // подтвердить код
P2PPong.getVerificationCode()         // получить код
P2PPong.getBeaconId()                 // получить beaconId
P2PPong.getPeerId()                   // получить peerId

// Обмен сообщениями
P2PPong.sendMessage(channelId, text)           // текстовое
P2PPong.sendVoiceMessage(channelId, audioB64)  // голосовое

// Профиль
P2PPong.setMyProfile(nick, avatar)
P2PPong.getTheirProfile()  // { nick, avatar }

// События
P2PPong.on('ready', () => {})
P2PPong.on('state-change', ({ state }) => {})                    // 'online' | 'offline' | 'connecting'
P2PPong.on('peer-id-generated', ({ peerId, beaconId, code }) => {})
P2PPong.on('verification-needed', ({ code }) => {})
P2PPong.on('verification-received', ({ code }) => {})
P2PPong.on('channel-opened', ({ channelId, peerId, nick, avatar }) => {})
P2PPong.on('channel-expired', ({ channelId }) => {})
P2PPong.on('peer-connected', ({ channelId, nick, avatar }) => {})
P2PPong.on('message-received', ({ channelId, text, voiceData, type, from, timestamp, nick, avatar }) => {})
P2PPong.on('message-sent', ({ channelId, data, status, nick, avatar }) => {})
P2PPong.on('beacon-timeout', () => {})
P2PPong.on('beacon-taken', () => {})
P2PPong.on('error', ({ message }) => {})
P2PPong.on('destroyed', () => {})
